`Visibility` 组件让单个页面服务两类受众:在网络上阅读的人类和处理 Markdown 输出的 AI 智能体。标记为 `for="humans"` 的内容会显示在渲染页面上,但在 Markdown 导出中被排除。标记为 `for="agents"` 的内容在网络上被隐藏,但包含在 Markdown 中。

这意味着您可以为人类编写对话式、以 UI 为中心的说明,为 AI 编写结构化、机器可读的说明——无需维护两个独立的页面。

## 工作原理

Mintlify 在两个 URL 上提供每个页面:

- `https://docs.example.com/page` — 渲染的网站,面向人类
- `https://docs.example.com/page.md` — 原始 Markdown,面向 AI 智能体

`Visibility` 控制在每个 URL 上显示的内容。

## 示例

```mdx
<Visibility for="humans">
  <Steps>
    <Step title="Open the dashboard">
      Navigate to [dashboard.example.com](https://dashboard.example.com) and sign in.
    </Step>
    <Step title="Create an API key">
      Go to **Settings → API Keys** and click **New key**.
    </Step>
  </Steps>
</Visibility>

<Visibility for="agents">
  To authenticate, send a POST request to `/v1/auth/token` with the user's
  email and password in the request body. The response includes a bearer token
  valid for 24 hours. Pass this token as `Authorization: Bearer <token>` on
  all subsequent requests.
</Visibility>
```

人类读者会看到逐步的 UI 演练。处理 Markdown 的 AI 智能体则获取直接的 API 说明。

## 属性

<ResponseField name="for" type='"humans" | "agents"' required>
  - `"humans"` — 在渲染的网页上可见,从 `.md` 导出中排除
  - `"agents"` — 在渲染的网页上隐藏,包含在 `.md` 导出中
</ResponseField>

## 与 Prompt 结合使用

`Visibility` 和 `Prompt` 可以很好地相互补充。使用 `Visibility` 给智能体提供结构化上下文,然后使用 `Prompt` 给人类读者一个提示,他们可以用它向 AI 询问相同的内容。

```mdx
<Visibility for="agents">
  ## Authentication schema
  - Method: Bearer token
  - Header: `Authorization: Bearer <token>`
  - Token TTL: 86400 seconds
  - Refresh endpoint: POST /v1/auth/refresh
</Visibility>

<Prompt description="Ask about authentication" icon="lock">
  Explain how authentication works in this API and show me example code for
  getting a token and making an authenticated request.
</Prompt>
```
